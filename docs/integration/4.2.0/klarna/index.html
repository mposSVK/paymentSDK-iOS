    Klarna¶

    Supported Transaction Types¶

    This payment method supports the following transaction types:

    • WDECTransactionTypeAuthorization - places the account holder's funds on hold, pending future capture, re-authorization or void transaction.

    Supported Klarna Payment Methods¶

    This payment method supports the following categories:

    • WDECKlarnaCategoryDebit - Klarna-debit for Klarna direct debit (Pay now). Type of Online Bank Transfer.
    • WDECKlarnaCategoryFinance - Klarna-finance for payment in installments (Financing/Slice it). Type of Offline Bank Transfer.
    • WDECKlarnaCategoryPayLater - Klarna-paylater for Klarna payment on invoive (Pay later). Type of Offline Bank Transfer.
    • WDECKlarnaCategoryTransfer - Klarna-transfer for Klarna bank transfer (Pay now). Type of Online Bank Transfer.

    **

    Note: Klarna takes full credit and fraud risk and guarantees payments. Klarna accepts only physical persons as consumers.

    **

    Basic Setup¶

    The Klarna module depends on the swift Klarna SDK framework. It offers support for the Swift 5.1 and for older versions too. Support for this is available through Cocoapods and by providing multiple builds made with several versions of Xcode as well as fat (device + simulator) builds.

    Variants¶

    For this release, we provide:

    Name Xcode Ver. & Build No. Swift Ver. Swift Toolchain Ver.

    Cocoapods¶

    The Klarna module is not a part of the All subspec. If you want to use the Klarna payment you need to add to your app's Podfile Klarna subspec. If you want to use a different variant (see tab above), update your Podfile to use a specific subspec, e.g:

    // 1. Initialize client object
// Parameters (hostname, merchantAccountID) may come from server to make it configurable.

do {
    try self.client = WDECClient.init(hostname: hostname)
} catch {
    // the device is checked during initialization for jailbreak.
    // It is risky to make payments on jailbroken devices.
    return
}

// 2. Initialize payment object and set mandatory properties
// Parameters (amount, currency, transactionType) shall be sent to server for validation or just provided by server
// and used to generate signature. Signature MUST be generated on server side.
// Signature doc: https://document-center.wirecard.com/display/PTD/Signature+v2
// Secret Key shall never be stored in application.

let payment = WDECKlarnaPayment()
payment.merchantAccountID = merchantAccountID  // provided by merchant support
payment.requestID = requestID                  // generated on server; unique for merchantAccountID
payment.amount = NSDecimalNumber(mantissa: 15, exponent: -2, isNegative: NO)
payment.currency = "EUR"
payment.locale = WDECLocale._de;
payment.transactionType = WDECTransactionType.authorizaton;
payment.category = WDECKlarnaCategory.payLater;
payment.appScheme = "yourapp.appscheme";
payment.signature = signature

let accountHolder = WDECCustomerData();
let address = WDECAddress()
address.country = WDECCountry.DE;
accountHolder.address = address;

payment.accountHolder = accountHolder;

var cartOrder = [WDECOrderItem]()

var orderItem = WDECOrderItem()
orderItem.articleNumber = "1"
orderItem.name = "Item 1"
orderItem.amount = NSDecimalNumber(integerLiteral: 20)
orderItem.quantity = NSDecimalNumber(integerLiteral: 1)
orderItem.amountCurrency = WDECCurrency.EUR
orderItem.type = WDECOrderItemType.physical

cartOrder.append(orderItem)

orderItem = WDECOrderItem()
orderItem.articleNumber = "2"
orderItem.name = "Item 2"
orderItem.amount = NSDecimalNumber(integerLiteral: 5)
orderItem.quantity = NSDecimalNumber(integerLiteral: 1)
orderItem.amountCurrency = WDECCurrency.EUR
orderItem.type = WDECOrderItemType.discount

cartOrder.append(orderItem)

let order = WDECOrder()
order.items = cartOrder

payment.order = order

// 3. Trigger the payment and handle response in payment completion block. 
self.client?.make(payment, withCompletion:{(response: WDECPaymentResponse?,error: Error?) in
    // for each failure error object is provided;
    guard let error = error else { return }
    if let response = response {
        self.handleResponse(response)
    }
})

    // 1. Initialize client object
// Parameters (hostname, merchantAccountID) may come from server to make it configurable.

NSError *error = nil;
self.client = [[WDECClient alloc] initWithHostname:hostname error:&error];
if (error) {
    // the device is checked during initialization for jailbreak.
    // It is risky to make payments on jailbroken devices.
    return;
}

// 2. Initialize payment object and set mandatory properties
// Parameters (amount, currency, transactionType) shall be sent to server for validation or just provided by server
// and used to generate signature. Signature MUST be generated on server side.
// Signature doc: https://document-center.wirecard.com/display/PTD/Signature+v2
// Secret Key shall never be stored in application.

WDECKlarnaPayment *payment = [WDECKlarnaPayment new];
payment.merchantAccountID = merchantAccountID;  // provided by merchant support
payment.requestID = requestID;                  // generated on server; unique for merchantAccountID
payment.amount = [NSDecimalNumber decimalNumberWithMantissa:15 exponent:-2 isNegative:NO];
payment.currency = @"EUR";
payment.locale = WDECLocale_de;
payment.transactionType = WDECTransactionTypeAuthorizaton;
payment.category = WDECKlarnaCategoryPayLater;
payment.appScheme = @"yourapp.appscheme";

WDECCustomerData *accountHolder = [WDECCustomerData new];
WDECAddress* address = [WDECAddress new];
address.country = WDECCountryDE;

accountHolder.address = address;

payment.accountHolder = accountHolder;

WDECOrderItem *order1 = [WDECOrderItem new];
order1.quantity = [NSDecimalNumber decimalNumberWithString:@"1"];
order1.name =  @"Item 1";
order1.articleNumber =  @"1";
order1.amount =  [NSDecimalNumber decimalNumberWithString:@"20"];
order1.amountCurrency = WDECCurrencyFromISOCode(@"EUR");
order1.type = WDECOrderItemTypePhysical;

WDECOrderItem *order2 = [WDECOrderItem new];
order2.quantity = [NSDecimalNumber decimalNumberWithString:@"1"];
order2.name =  @"Item 2";
order2.articleNumber =  @"2";
order2.amount =  [NSDecimalNumber decimalNumberWithString:@"5.0"];
order2.amountCurrency = WDECCurrencyFromISOCode(@"EUR");
order2.type = WDECOrderItemTypeDiscount;

payment.order = [WDECOrder new];
payment.order.items = @[order1, order2];

payment.signature = signature;

// 3. Create payment completion block to handle response

@weakify(self);
WDECCompletionBlock completionBlock = ^(WDECPaymentResponse * _Nullable response, NSError * _Nullable error) {
    @strongify(self);
    // for each failure error object is provided;
    if (error) {
        return;
    }
    [self handleResponse:response];
}

// 4. Trigger the payment
[self.client makePayment:payment withCompletion:completionBlock];

    Note: You can set optional parameters after the payment object is created. Check the API documentation:WDECKlarnaPayment for more details.
